Encryption

What is Encryption?

Encryption is the method of a readable data called "plaintext" into different format, which can be only understood by the authorized person, in more simple way, it's the operation of converting plaintext into cipher text using mathimcatical operations. Encryption is a reversable operation, so it can be reversed to get the plaintext again, and this operation is called decryption.

Encryption algorithms uses encryption key for the encryption process, which has to be owned by both sender and receiver. It's used while encryption and decrption.

Cipher text seems to be random and sure unreadable, but it's the output of mathimatical operations, and also predictable, So each time the same algorithm is used to encrypt the same plaintext using the same encryption key, will generate the same cipher text.



Encryption Algorithm

The mathematical formula used for transforming data into ciphertext is an encryption algorithm. An algorithm will use the key to predictably alter the data, so that even if the encrypted data appears random, it can be turned back into plaintext again using the key.



Why Encryption is needed?

Encryption is used to fulfill some security aspects:

  • Privacy: Encryption ensures no one, except the intended recipient or proper data owner, can read messages or data at rest. It avoids the collection and reading of confidential data by cyber criminals, ad networks, Internet service providers and even governments.

  • Security: Encryption supports in case of theft of data, whether the data is in transit or in rest. When a corporate computer is lost or stolen, and its hard drive is correctly secured, it is possible that the data on that computer will remain secure. Similarly, encrypted communications allow the communicating parties to share confidential information without leaking the data. Encryption also helps prevent malicious activities like Man In The Middle (MITM) from occurring.

  • Authentication: The public key encryption establishes, among other things, that the origin server of a website owns the private key and therefore has been given an SSL certificate legitimately

  • Regulations: Most industry and government laws allow organizations that manage user data to keep their data encrypted for all of these purposes. HIPAA, PCI-DSS, and the GDPR are examples of regulatory and compliance standards that require encryption.

Symmetric vs Asymmetric

The major difference between these two encryption methods is that asymmetric encryption algorithms use two distinct but related keys — one key to encrypt the data, and another key to decrypt it — while symmetric encryption uses the same key to both encryption and decryption functions.


In symmetric cryptography, the key length — randomly chosen — is usually set at 128-bit or 256-bit depending on the degree of protection required.


Asymmetric encryption is slower than symmetric encryption, with a higher speed of execution.
